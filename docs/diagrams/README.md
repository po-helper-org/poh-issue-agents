# Диаграммы процесса

Полный бизнес-процесс разбит на две части (для читаемости на экране).
Оба файла — Mermaid, рендерятся нативно на GitHub при вставке в `.md`.

## Легенда цветов
- Зелёная обводка — реализовано в заготовке (предфильтры, gate,
  классификация, дубликаты, приоритизация).
- Синяя — точки решения человека (ответ на уточнение, запуск пайплайна,
  проверка артефактов, ревью PR).
- Жёлтая пунктирная — стадия кода (OpenHands/pr-agent/deploy), вне ядра.
- Красная — терминальные остановки (спам/бот/security/дубликат/эскалация).

## Часть 1 — intake (issue-workflow-part1-intake.mermaid)
От "пользователь открыл Issue" до расчёта приоритета: предфильтры → intake
gate → классификация → duplicate check → priority scoring. Всё автономно,
ни одного платного research-вызова.

## Часть 2 — pipelines (issue-workflow-part2-pipelines.mermaid)
От решения человека после приоритизации до деплоя: bug/research-пайплайны →
build-me → OpenHands/pr-agent → merge → capabilities.

Стык: узел с приоритетом P0-P3 в конце части 1 = вход в часть 2.
