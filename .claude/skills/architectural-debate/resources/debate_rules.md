# Правила проведения дебатов

## Общие правила

1. Дебаты проводятся в 3 раунда. Количество раундов фиксировано.
2. Каждый раунд чередует: Архитектор → Адвокат Дьявола.
3. Модератор (LLM) не занимает сторону — обеспечивает структуру.
4. Дебаты записываются в конец файла concept.md.

## Раунд 1: Защита

**Архитектор (выбор + обоснование):**

- Выбирает лучший концепт из предложенных.
- Формулирует 3-5 аргументов «за».
- Каждый аргумент подтверждён кодом.

**Адвокат Дьявола (атака):**

- Оспаривает аргументы Архитектора.
- Формулирует 3-5 контраргументов.
- Каждый контраргумент конкретен и привязан к реальным рискам.

**Чек-лист проверки Архитектора:**

- [ ] Выбран конкретный концепт (не «комбинация»)
- [ ] Минимум 3 аргумента
- [ ] Каждый аргумент имеет доказательство из кода

**Чек-лист проверки Адвоката:**

- [ ] Минимум 3 контраргумента
- [ ] Каждый контраргумент конкретен (не «это плохая практика»)
- [ ] Рассмотрены: нагрузка, отказ компонента, изменение требований, масштабирование

## Раунд 2: Пересмотр

**Архитектор (ответ):**

- Отвечает на каждый контраргумент Адвоката.
- Усиливает концепт или пивотит на другой.
- Если пивотит — формулирует, какой концепт и почему.

**Адвокат Дьявола (поверка):**

- Проверяет усиленный вариант на новые слабости.
- Задаёт вопрос: «А что если условие X изменится через 6-12 месяцев?».

**Допустимые исходы раунда:**

- Архитектор усилил концепт → переход к раунду 3.
- Архитектор пивотил → Адвокат проверяет новый концепт.
- Архитектор не смог ответить на критику → вердикт «забраковано» или «пивот».

## Раунд 3: Вердикт

**Архитектор:** Итоговая позиция (1-2 предложения).

**Адвокат Дьявола:** Итоговая позиция (1-2 предложения).

**Вердикт формируется модератором:**

| Тип вердикта | Условие |
|-------------|---------|
| Концепт принят | Обе стороны согласны, критика адресована |
| Принят с модификациями | Архитектор усилил концепт, Адвокат признал усиление достаточным |
| Пивот | Архитектор перешёл на другой концепт, Адвокат согласен |
| Забраковано | Адвокат выявил неприемлемые риски, Архитектор не смог их митигировать |
| Не техническая проблема | Обе стороны согласны, что проблема процессная/организационная |

## Запреты

1. Запрещено формулировать вердикт до завершения раунда 3.
2. Запрещено игнорировать контраргументы Адвоката.
3. Запрещено фабриковать доказательства — только реальный код.
4. Запрещено менять субъект дебатов (переходить на другую проблему).
