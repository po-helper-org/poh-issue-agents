# Чек-лист валидации task.md

## 1. Полнота постановки

- [ ] Проблема сформулирована в 1-2 предложениях, без двояких толкований
- [ ] Контекст описывает, как проблема была обнаружена и при каких условиях проявляется
- [ ] Тип задачи определён (баг / рефакторинг / фича / оптимизация / архитектурное изменение)

## 2. Фактологическая точность

- [ ] Все описания поведения подтверждены код-доказательствами (файл:строка)
- [ ] [CRITICAL] Нет утверждений без доказательств? (Если есть — пометить `[NEEDS_INVESTIGATION]`)
- [ ] Цепочка «симптом → корень» прослежена полностью, без пропусков

## 3. Корень проблемы

- [ ] Выделен корень проблемы (не симптом)
- [ ] Корень подтверждён минимум одним код-доказательством
- [ ] [CRITICAL] Не перепутаны корень и симптом

## 4. Зона воздействия

- [ ] Перечислены все компоненты прямого воздействия
- [ ] Перечислены зависимые компоненты (косвенное воздействие)
- [ ] [CRITICAL] Зафиксированы компоненты, которые НЕ должны быть затронуты

## 5. Отсутствие решений

- [ ] [CRITICAL] В документе нет предложенных решений — только постановка проблемы
- [ ] Нет оценок подходов («лучше», «правильнее», «быстрее»)
- [ ] Нет требований («система должна») — только описание текущего и ожидаемого поведения

## 6. Ссылки и пути

- [ ] Все ссылки на файлы — относительные (нет абсолютных путей)
- [ ] Указаны ссылки на существующую документацию, если релевантна
- [ ] Используется терминология из naming_conventions.md (если файл существует)

## 7. Структура

- [ ] Документ содержит все 8 обязательных разделов шаблона

---

## Hard Gates (🔴)

1. **Root Cause Gate:** Не выделен корень проблемы — 🔴.
2. **Evidence Gate:** Утверждение о поведении без доказательства (файл:строка) — 🔴.
3. **No Solution Gate:** Документ содержит предложенные решения — 🔴.
4. **Path Gate:** Абсолютные пути в ссылках — 🔴.
5. **Template Gate:** Пропущен любой из обязательных разделов шаблона — 🔴.
