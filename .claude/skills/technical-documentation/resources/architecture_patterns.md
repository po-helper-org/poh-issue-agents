# Универсальная карта архитектурных паттернов (Universal Patterns)

Используй эти правила для классификации любого кода и его визуализации на диаграммах C4 Level 3.

## 1. Слой Обработки Входа (Ingress Handler / Routing)
- **Признаки:** Наличие сетевых аннотаций (`@Route`, `@Controller`, `@Endpoint`), реализация интерфейсов слушателей очередей или обработчиков команд CLI.
- **Роль:** Десериализация транспорта, базовая валидация, маршрутизация вызова вглубь системы.
- **В схеме:** `Component` с ролью "Entry Point".

## 2. Слой Бизнес-Домена (Logic / Orchestration)
- **Признаки:** Манипуляция сущностями проекта, отсутствие прямой зависимости от HTTP/SQL. Названия часто включают `*Service`, `*UseCase`, `*Processor`.
- **Роль:** Реализация бизнес-правил, оркестрация вызова нескольких репозиториев или внешних систем.
- **В схеме:** `Component` с ролью "Domain Logic".

## 3. Слой Персистентности (Persistence / Storage)
- **Признаки:** Использование ORM, выполнение Raw-запросов, реализация интерфейсов `Repository` или `DAO`.
- **Роль:** Маппинг данных из кода в физическое хранилище (SQL, NoSQL, Key-Value).
- **В схеме:** `Component` с ролью "Storage Adapter".

## 4. Слой Внешних Интеграций (Integration Gates)
- **Признаки:** Использование сетевых клиентов (HTTP Client, RPC, SOAP, MQ Producers).
- **Роль:** Инкапсуляция протоколов связи с внешними системами.
- **В схеме:** `Component` с ролью "Integration Bridge".

# Интеллектуальный поиск сигнатур (Heuristics)

Если ты встречаешь следующие паттерны, сопоставляй их с архитектурными ролями:

### 1. Сигналы Интеграций (External Context)
- **Паттерн:** Использование специфических URL-паттернов или библиотек связи.
- **Пример:** URL-префикс `/api/v1/payments/` -> Gateway к платёжному сервису; URL `/internal/sync/` -> синхронизация с микросервисом.
- **Действие:** Отрисовывай как `Container` внешней системы в `Boundary` этой системы.

### 2. Сигналы Инфраструктуры (Infrastructure)
- **Паттерн:** Классы для работы с кэшем, очередями, поисковыми движками.
- **Пример:** `SearchClient` -> Search Engine; `CacheManager` -> KV-Store.
- **Действие:** Отрисовывай как инфраструктурный бокс, отмечай протокол связи.

### 3. Сигналы Наследования (Framework Inheritance)
- **Паттерн:** Наследование от базовых классов фреймворка или использование аннотаций.
- **Пример (Legacy):** `extends BaseController` -> Роль Ingress; `extends BaseModel` -> Роль Domain Entity / Logic.
- **Пример (Modern):** `@RestController` -> Роль Ingress; `@Entity` -> Роль Domain Entity.

---
**МЕТОДОЛОГИЯ:** Начинай анализ любого нового проекта с поиска файлов конфигурации (например, `application.properties/yaml/json`, `routes`, `config`) и структуры директорий, чтобы определить «карту сигнатур» данного конкретного стека.
