## 1. Фактологическая точность

- [ ] Соответствуют ли типы данных (String, Numeric, Boolean, Object) реализации в коде?
- [ ] Существует ли указанный эндпоинт/обработчик в конфигурации роутинга проекта?

## 2. Отсутствие галлюцинаций

- [ ] [CRITICAL] Есть ли в тексте утверждения, которые НЕЛЬЗЯ подтвердить кодом? (Если да — исправить или пометить [NEED_CHECK]).
- [ ] Соответствует ли описанный алгоритм реальному потоку управления в методах (без додумывания логики)?

## 3. Соответствие стандартам

- [ ] Соблюдена ли структура согласно целевому шаблону (напр. `ideal_api_document.md`)?
- [ ] [CRITICAL] Для каждого "листа" данных (конечного поля) указано физическое происхождение (Store.Entity.Attribute)?
- [ ] [CRITICAL] **Full Entity Coverage:** Сверены ли ВСЕ поля из программного класса ($schema/DTO/Entity) с таблицей маппинга? Отсутствие полей без обоснования — 🔴.

## 4. Проверка инженерной глубины

- [ ] Если данные поступают из внешнего сервиса, указан ли конкретный атрибут из ответа этого сервиса?
- [ ] Описаны ли условия ветвления (`if/else`, `switch`), влияющие на состав выходных данных?
- [ ] Отражены ли в UML-диаграмме вызовы к слою персистентности (напр. `Repository::find`, `Storage::get`)?

## 5. Валидация интерфейса и контракта

- [ ] Описан ли процесс трансформации входящих параметров (напр. из Alias в ID), если он есть в коде?

- [ ] Указаны ли значения по умолчанию для необязательных аргументов?

## 6. Автономность и Полнота

### Hard Gates (Условия СТРОГОГО 🔴)

1. **Hallucination Gate:** Обнаружено хотя бы одно поле или метод, не существующий в коде.
2. **Lineage Gate:** Отсутствует маппинг на физическую таблицу/колонку для любого из выходных полей.
3. **Visualization Gate:** В UML-диаграмме отсутствует блок `note` с примером структуры данных (JSON/Array).
4. **Coverage Gate:** В документе пропущено более 10% полей из исходного программного объекта (DTO/Entity) без явного обоснования их исключения.
5. **Language Gate:** Надписи на стрелках UML выполнены не на русском языке или содержат только технические имена методов.
6. **Path Gate:** В документе обнаружены абсолютные пути (`file:///`). Все ссылки должны быть относительными.
