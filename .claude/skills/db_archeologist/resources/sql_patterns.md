# Справочник паттернов доступа к данным

## 1. Паттерны фильтрации (Business Flags)
Используй эти сопоставления при описании логики:
- `is_active = 1` -> "Фильтр активных/валидных записей".
- `is_deleted = 0` -> "Исключение удаленных (soft delete) объектов".
- `tenant_id IN (...)` -> "Ограничение видимости в рамках конкретного клиента/партнера".

## 2. Паттерны идентификации
- `id` vs `nItemId` vs `item_id`: Если в коде используются разные именования для одного и того же ключа, зафиксируй это в отчете как "Alias Mapping".

## 3. Специфика СУБД
- **Views (Представления):** Если запрос идёт к view (представлению), помечай это как "Агрегированная витрина данных" (обычно объединяет несколько таблиц).
- **Linked Servers / Cross-database queries:** Если в SQL видишь формат `[DB].[Schema].[Table]`, это признак межсерверного запроса.

## 4. Паттерны кэширования
- Если перед запросом к БД идёт проверка ключа в кэше, помечай это как `DoubleCache Pattern`.
