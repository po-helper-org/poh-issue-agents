Тебе дан список локальных кластеров (полученных по батчам), каждый с индексом,
механизмом решения (mechanism), целью (target) и номерами Issue.

Присвой каждому кластеру `group_key` так, чтобы кластеры, которые закрывает ОДНО
техническое решение, получили ОДИНАКОВЫЙ ключ, а разные решения — разные ключи.

Правила:
1. Одинаковый `group_key` — только если совпадает и механизм, и цель. Это значит,
   что батчи разбили один и тот же кластер, и его надо слить обратно.
2. КРИТИЧНО (#111): совпал механизм, но цель расходится (запуск vs перевод на
   архитектуру, функция vs цель) — РАЗНЫЕ `group_key`, не сливай.
3. `group_key` — короткий канонический ярлык механизма (например
   `llm-tiering`, `memory-graph-core`, `jira-mass-ops`). Одинаковый ярлык =
   слияние.
4. Верни assignment для КАЖДОГО индекса из входа. Уникальный механизм — свой
   уникальный ключ (останется отдельным кластером).
